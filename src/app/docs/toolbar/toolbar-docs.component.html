The toolbar component displays a SKY UX-themed toolbar. Use the sky-toolbar component in conjunction with the sky-toolbar-item child component.

The list toolbar appears above a list of records. It contains actions to manipulate the contents of the list. Place frequently used actions directly in the toolbar, and place less-frequently used actions in a more actions menu. When multiple views of the list are available, the view switcher is docked on the right side of the toolbar.

The record toolbar appears below the page summary on record pages. It contains actions to manipulate the record. Place frequently used actions directly in the toolbar, and place less-frequently used actions in a more actions menu. Actions to collapse and expand all tiles on the record are docked on the right side of the toolbar.

The tile toolbar appears in a tile above the control it affects. It contains actions to manipulate the contents of the tile or a portion of the tile. Only place the most important actions directly in the toolbar because tiles have limited horizontal real estate. Place all other actions in a more actions menu. If a tile requires no more than two filters, the filters should be docked on the right side of the toolbar.

The actions in a toolbar (including the more actions button) shouldn't extend beyond 50% of the screen. If the actions would extend beyond 50% of the screen, the rightmost action should collapse into the more actions menu.

If a toolbar doesn't include a more actions menu and the actions extend beyond 50% of the screen, create a more actions menu for the two rightmost actions.

On small screens, action buttons should switch to icon-only buttons, the search field should collapse into a search icon, and the view switcher should collapse into a single icon. Buttons without icons should go in the more actions menu, and the more actions button, view switcher, and any other buttons with dropdowns should display their options when pressed.
